# PlayingDice

Визуализация игральных кубиков на микроконтроллере GD32F303CCT6 с цветным IPS-дисплеем 0,96" (80×160, ST7735S). Проект собирается в **IAR Embedded Workbench**.

## Железо

| Компонент | Описание |
|-----------|----------|
| МК | GD32F303CCT6 (Cortex-M4, 120 МГц, 256 КБ Flash, 48 КБ RAM) |
| Дисплей | 0,96" IPS RGB 80×160, контроллер ST7735S |
| Интерфейс дисплея | SPI (4-wire: SCK, MOSI, CS, DC + RES, BLK) |
| Кнопки | 2 шт., тактовые, на землю (активный уровень — LOW) |

---

## Распиновка дисплея

Дисплей имеет 8 контактов. Ниже — их назначение и куда подключать на МК:

| Контакт дисплея | Функция | Пин МК (по умолчанию) | Зачем |
|---|---|---|---|
| **GND** | Земля | GND | Общая земля |
| **VCC** | Питание | 3.3 В | Питание дисплея |
| **SCL** | SPI Clock (тактовый) | **PA5** (SPI0_SCK) | Тактирует передачу данных |
| **SDA** | SPI MOSI (данные) | **PA7** (SPI0_MOSI) | Данные от МК к дисплею |
| **RES** | Hardware Reset | **PB1** | Аппаратный сброс дисплея при старте |
| **DC** | Data / Command | **PB0** | LOW = команда, HIGH = данные пикселей |
| **CS** | Chip Select | **PA4** | Активирует дисплей на шине SPI |
| **BLK** | Backlight | **PB10** | Управление подсветкой (вкл/выкл) |

> На этих дисплеях контакты **SCL/SDA** — это SPI, а не I²C, несмотря на названия.

## Распиновка кнопок

Кнопки подключаются между пином МК и **GND**. Внутренние подтяжки к VCC включаются программно (`GPIO_MODE_IPU`), поэтому внешние резисторы не нужны.

| Кнопка | Пин МК (по умолчанию) | Функция |
|---|---|---|
| **BTN1** | **PC13** | Бросить кубики / запустить новую анимацию |
| **BTN2** | **PC14** | Короткое нажатие — режим "≤8"; долгое (>0,6 с) — 1 или 2 кубика |

## Где менять пины

Все пины вынесены в один файл — **`st7735.h`**, секция `Pin mapping`:

```c
/* ---- Pin mapping — adjust to your board ---- */

/* SPI0: PA5 = SCK (SCL), PA7 = MOSI (SDA) — менять в LCD_Init() в st7735.c */
#define LCD_CS_PORT   GPIOA
#define LCD_CS_PIN    GPIO_PIN_4

#define LCD_DC_PORT   GPIOB
#define LCD_DC_PIN    GPIO_PIN_0

#define LCD_RES_PORT  GPIOB
#define LCD_RES_PIN   GPIO_PIN_1

#define LCD_BLK_PORT  GPIOB
#define LCD_BLK_PIN   GPIO_PIN_10

/* Кнопки */
#define BTN1_PORT     GPIOC
#define BTN1_PIN      GPIO_PIN_13

#define BTN2_PORT     GPIOC
#define BTN2_PIN      GPIO_PIN_14
```

> **SCK и MOSI** жёстко привязаны к аппаратному SPI0 (PA5, PA7) и задаются в функции `LCD_Init()` в `st7735.c`. Если нужен другой SPI — меняй там же и номер периферии (`SPI0` → `SPI1` и т.д.).

### Скорость SPI

В `st7735.c`, функция `LCD_Init()`:

```c
sp.prescale = SPI_PSC_4;   /* 120 МГц / 4 = 30 МГц */
```

Если дисплей не откликается — попробуй `SPI_PSC_8` (15 МГц) или `SPI_PSC_16` (7,5 МГц).

### Смещение адресов дисплея

Некоторые партии 80×160 ST7735S имеют физическое смещение матрицы. Если картинка сдвинута — найди в `st7735.c` функцию `LCD_SetWindow` и добавь оффсет:

```c
/* Пример: COLSTART = 26, ROWSTART = 1 (типично для ряда панелей) */
lcd_dat((x0 + 26) >> 8);  lcd_dat((x0 + 26) & 0xFF);
lcd_dat((x1 + 26) >> 8);  lcd_dat((x1 + 26) & 0xFF);

lcd_dat((y0 + 1) >> 8);   lcd_dat((y0 + 1) & 0xFF);
lcd_dat((y1 + 1) >> 8);   lcd_dat((y1 + 1) & 0xFF);
```

## Функционал

```
Включение / нажатие BTN1
        │
        ▼
  "by Xelor" — 1 секунда
  (белый фон, «by» чёрным, «Xelor» цветом морской волны)
        │
        ▼
  Кубики вылетают с краёв экрана
  — физика: ease-out квадратичная, ~3 секунды
  — вращение граней: быстро → медленно → стоп
        │
        ▼
  Результат: числа над кубиками
  (красным — кубик 1, белым — кубик 2, итог — морская волна)
  Кубики остаются на экране
        │
        ├── BTN1 → новый бросок с анимацией
        ├── BTN2 → смена режима (работает немедленно)
        └── 20 секунд без нажатий → экран гаснет, МК в режиме сна
                    │
                    └── Любая кнопка → выход из сна, возврат в ожидание
```

## Режимы игры

### Иконки в углах (всегда видны)

```
┌─────────────────────────────────────┐
│ [патрон]              [■] [■]       │  ← статус-бар (14 px)
│                                     │
│        [кубик 1]   [кубик 2]        │
└─────────────────────────────────────┘
```

| Иконка | Расположение | Смысл |
|---|---|---|
| Патрон (бронзовый) | Верхний левый угол | Режим "не более 8" активен |
| Квадрат(ики) | Верхний правый угол | Количество кубиков: 2 квадрата = 2 кубика, 1 квадрат = 1 кубик |

### Кнопка 2 — управление режимами

| Действие | Эффект |
|---|---|
| Короткое нажатие | Вкл/выкл режим **"≤8"** — сумма кубиков не превысит 8 |
| Удержание >0,6 с | Переключение **1 ↔ 2 кубика** |

> Во время анимации BTN2 игнорируется. Режим фиксируется до начала броска.

### Режим одного кубика

- Кубик вылетает случайно: слева **или** справа.
- Результат пишется по центру белым цветом.
- Иконка в правом углу — один квадрат.

## Сборка в IAR

1. Открыть `test.eww` в IAR Embedded Workbench.
2. Убедиться, что в настройках проекта → **C/C++ Compiler → Preprocessor** задан символ `GD32F30X_HD`.
3. Выбрать конфигурацию **Debug** или **Release**.
4. **Project → Make** (F7).
5. Прошить через **Download and Debug** или J-Link Commander.

> Файл `main.c` самостоятельно добавляет `#define GD32F30X_HD` если он не задан снаружи — это нужно только для корректной работы IntelliSense в VS Code и не влияет на IAR.

## Типичные проблемы

| Симптом | Причина | Решение |
|---|---|---|
| Экран белый / чёрный, нет изображения | Неверные пины или SPI не инициализирован | Проверь `st7735.h`, подачу 3,3 В на VCC и BLK |
| Картинка смещена или обрезана | Оффсет матрицы | Добавь COLSTART/ROWSTART в `LCD_SetWindow` (см. выше) |
| Картинка перевёрнута / зеркальная | Неверный MADCTL | Измени байт `0x36` в `LCD_Init`: `0x60`, `0xA0`, `0xC0` или `0x00` |
| Артефакты, полосы | Слишком высокая скорость SPI | Уменьши делитель: `SPI_PSC_8` вместо `SPI_PSC_4` |
| Кнопки не реагируют | Неверный порт/пин или активный уровень | Проверь `BTN1_PORT`/`BTN1_PIN` в `st7735.h`; убедись что кнопка замыкает на GND |
| Сон не работает / не выходит из сна | PC13/PC14 не подключены к EXTI | Кнопки должны быть именно на PC13/PC14 для EXTI10_15; при смене пинов — обнови `EXTI10_15_IRQHandler` |
| `EXTI10_15_IRQn` подчёркнуто в VS Code | Ложная ошибка IntelliSense | Нормально, IAR компилирует без ошибок — символ есть в `gd32f30x.h` под `GD32F30X_HD` |

## Цвета (RGB565)

| Константа | Hex | Описание |
|---|---|---|
| `COLOR_BLACK` | `0x0000` | Фон |
| `COLOR_WHITE` | `0xFFFF` | Кубик 1 (тело), точки кубика 2, текст |
| `COLOR_RED` | `0xF800` | Точки кубика 1, цифра кубика 1 в счёте |
| `COLOR_TEAL` | `0x0596` | «Xelor», итоговая сумма (~#00B2AA) |
| `COLOR_BRONZE` | `0xCBE6` | Иконка патрона (~#CD7F32) |
| `COLOR_DKGRAY` | `0x4208` | Тело кубика 2 |

Менять в `st7735.h`.
